## 앱 구조

```
App.tsx
├── TabBar (하단 네비게이션: 홈 / 캘린더 / 설정)
├── Home (홈 탭)
├── CalendarPage (캘린더 탭)
└── Settings (설정 탭)
```

**상태 관리**: 전역 상태 없음. 모든 데이터는 `useCalendar` 훅을 통해 Firestore와 직접 실시간 동기화.

**탭 전환**: 조건부 렌더링(`{tab === 'x' && <Component />}`) → 탭 이동 시 컴포넌트 언마운트/재마운트됨.

## 홈 탭

**파일**: `src/pages/Home.tsx`

### 표시 조건별 로직

| 조건 | 오늘 패널 표시 내용 |
|------|---------------------|
| 비번 (전날 당직 아님) | "🏖️ 비번" 텍스트 |
| 비번 + 전날 당직 + 오전(~11시) | 퇴근 `09:00` + 귀가도착 시간 |
| 당직 + 오전 | ⏰기상 / 🚗출발시간 / 🌙익일퇴근(+귀가소요) |
| 당직 + 오후 | 🌙내일퇴근 `09:00` + 🏠귀가도착 |
| 교번 + 오전 | ⏰기상 / 🚗지금출발→도착예정 |
| 교번 + 오후 | 🏢퇴근예정(+도착) / 🚗지금출발→귀가도착 |

**오전/오후 기준**: `today.getHours() < 11` (11시 기준, 분 단위 미적용)  
**isPostNightShift**: `todayShift === '비' && yesterdayShift === '당'`

### 날씨

- API: Open-Meteo (`forecast_days=2`, 무료)
- 오늘 기온·날씨코드·풍속·강수확률 + 내일 예보
- 회사 위치 좌표 기반 (`settings.workplaceLat/Lng`)
- 위치 미설정 시 서울(37.5665, 126.9780) 기본값으로 조용히 대체
- 옷차림: 기온 구간별 텍스트 (`src/utils/weather.ts` → `getOutfitShort`)
- 강수확률 ≥ 50%이면 "우산 챙기세요" 뱃지 표시
- 풍속 ≥ 7m/s이면 "바람 강함" 뱃지 표시

### 출퇴근 소요시간 (Tmap)

- 출발지 탭에서 위치 선택 시 `getTmapDuration` 호출
- **출근 소요시간**: 출발지 → 회사 (도착목표 기준 1시간 전 출발로 계산)
- **귀가 소요시간**: 회사 → 출발지
  - 당직/당직다음비번: `09:00` 기준
  - 일반 교번 오후: 퇴근시간 기준
- 기본 출발지(`defaultLocationId`) 설정 시 앱 열 때 자동 선택

### 이번 주 미니 캘린더

- 오늘 포함 7일 표시
- 날짜별 근무뱃지 + 첫 번째 메모 미리보기(7자)
- 오늘 강조: 파란 테두리(`inset box-shadow`)

### 아 맞다! (할일)

- 텍스트 추가/완료 토글/삭제
- Firestore `todos` 배열에 즉시 저장
- `id`: `Date.now().toString()` (밀리초 타임스탬프)

## 캘린더 탭

**파일**: `src/pages/CalendarPage.tsx`, `src/components/MonthCalendar.tsx`, `src/components/DayBottomSheet.tsx`

### 월별 뷰

- 이전/다음 월 이동 버튼
- 날짜 셀: 날짜 숫자 + 공휴일명 + 근무뱃지 + 메모 미리보기(최대 2개)
- 수동변경(오버라이드)된 날: 날짜 숫자 우상단 빨간 점
- 오늘: 파란 테두리
- 일요일·공휴일: 빨간색, 토요일: 파란색

### 캡처 모드 (강아지 버튼)

- 토글 ON: 모든 메모를 "개인일정"으로 마스킹
- 목적: 스크린샷 공유 시 개인 일정 노출 방지
- 강아지 이미지 변경(`dog1` ↔ `dog2`)으로 상태 표시

### DayBottomSheet (날짜 탭)

**메모 탭** (기본):
- 메모 추가/수정/삭제
- 수정 시 입력창에 기존 텍스트 로드, 완료 시 해당 위치 덮어쓰기
- 최대 30자, 개수 제한 없음
- 변경 즉시 Firestore 저장

**근무 탭**:
- 6가지 선택지: 교1 / 교2 / 당 / 비 / 전진배치 / 중요업무
- "자동" 선택 시 오버라이드 삭제 (패턴 복원)
- 현재 자동 근무 표시: "본근무" 라벨
- 변경 즉시 Firestore 저장

### 월별 메모

- textarea, blur 시 Firestore 저장
- 키: `YYYY-MM` 형식
- 캡처 모드 시 "개인일정"으로 마스킹

### 근무 계산 엔진 (`src/utils/shiftEngine.ts`)

```
사이클: [교1, 교2, 당, 비] (4일 순환)
규칙: 교1·교2가 주말(토·일) 또는 한국 공휴일이면 → 비
오버라이드: Firestore 저장값이 계산값보다 항상 우선
```

- `date-holidays` 라이브러리로 공휴일 판단
- 시작일 이전 날짜도 역방향 계산 지원

## 설정 탭

**파일**: `src/pages/Settings.tsx`

### 자동 저장

- 입력 변경 후 1초 디바운스로 Firestore 저장
- 저장 완료 시 "✓ 저장됨" 표시 (1.5초)
- 최초 로드 시 불필요한 저장 방지(`isFirstLoad` ref)

### 근무 패턴

- 기준 시작일 (date picker)
- 첫 근무 유형 선택: 교1 / 교2 / 당 / 비

### 근무별 시간 설정

각 근무(교1·교2·당·전진배치·중요업무)별:
- **도착목표**: 출근 목표 도착 시간 → 역산으로 출발시간 계산
- **기상**: 홈화면 기상 알림 시간 표시용
- **퇴근**: 홈화면 오후 퇴근 예정 시간 표시용

### 회사 위치

- 회사 이름 입력
- 주소 입력 → 검색(Nominatim API) → 좌표 변환 → "✓ 위치 확인됨"
- 저장된 좌표로 날씨·경로 계산

### 추가 출발지 (최대 3개)

- 별명 + 주소 입력/검색
- 기본 출발지 설정: ★ 표시, 홈화면 자동 선택
- 삭제 시 기본 출발지였으면 `defaultLocationId` 자동 해제

### 알림 설정

| 상태 | 표시 |
|------|------|
| 미허용(default) | "알림 허용하기" 버튼 |
| 거부(denied) | "기기 설정에서 허용해주세요" 안내 |
| 허용(granted) | ON/OFF 토글 |

- 허용 시: FCM 토큰 발급 → Firestore `fcmTokens` 저장 → 기존 다른 토큰 자동 정리
- OFF 시: Firestore 토큰 삭제
- iOS: 홈화면 추가 후 PWA 모드에서만 작동 (iOS 16.4+)
- 비지원 환경: "이 브라우저는 알림을 지원하지 않습니다" 에러 표시

### 공유

- "공유 링크 복사": `{origin}?cal={calendarId}` 클립보드 복사
- 링크 접속 시 `?cal=` 파라미터 자동 인식 → 해당 캘린더 로드 및 localStorage 저장

## 당직 리마인더 (푸시 알림)

**파일**: `scripts/duty-reminder.mjs`, `.github/workflows/duty-reminder.yml`

### 트리거 구조

```
cron-job.org (스케줄러)
  → HTTP POST → GitHub API (workflow_dispatch)
  → GitHub Actions 실행
  → duty-reminder.mjs 실행
  → Firestore 조회 + FCM 발송
```

### 스케줄

| 시간 (KST) | mode | 대상 | 메시지 |
|-----------|------|------|--------|
| 17:50 | duty1 | 오늘 `당` 근무인 캘린더 | 📋 10분 후 근무일지 |
| 08:10 | duty2 | 어제 `당` 근무였던 캘린더 | 🐕 견사정비 이상유무 |

### 로직

1. Firestore `calendars` 컬렉션 전체 조회
2. 각 캘린더의 `settings.startDate`, `settings.startShift`, `overrides`로 대상일 근무 계산
3. 근무 조건 일치 → `fcmTokens` 서브컬렉션에서 토큰 수집
4. 토큰 중복 제거 (Map 사용)
5. `sendEachForMulticast` 발송
6. 실패 토큰(만료·무효) 자동 삭제

### test 모드

근무 조건 없이 토큰 있는 모든 캘린더에 테스트 발송. GitHub Actions → Run workflow → mode: test 로 수동 실행.

## 공유 기능

- 설정 탭 → "공유 링크 복사"
- URL: `https://shift-calendar-6945f.web.app?cal={calendarId}`
- 링크 접속 시 `?cal=` 파라미터 인식 → localStorage에 저장 → 해당 캘린더 사용
- **주의**: 링크를 가진 누구나 읽기·쓰기 가능 (인증 없음)

## 데이터 흐름

```
useCalendar 훅
  ├── localStorage: shift_calendar_id (캘린더 ID 저장)
  ├── Firestore onSnapshot: 실시간 구독 (탭 전환 시 재구독)
  └── update(): setDoc merge로 부분 업데이트

FCM
  ├── localStorage: fcm_token (현재 기기 토큰 캐시)
  └── Firestore: calendars/{id}/fcmTokens/{token}
```

## 알려진 제한사항

| 항목 | 내용 |
|------|------|
| 알림 지연 | cron-job.org 자체 지연 가능 (보통 1분 이내) |
| iOS 알림 | 홈화면 추가 + PWA 모드 필수 (iOS 16.4+) |
| 탭 재마운트 | 탭 이동 시 컴포넌트 재생성 → Tmap API 재호출 |
| 날씨 오류 무음 | API 실패 시 "날씨 로딩..." 영구 표시 |
| Firestore 보안 | 인증 없음, calendarId로만 보호 |
| 오프라인 미지원 | 앱 셸 캐싱 없음 |
